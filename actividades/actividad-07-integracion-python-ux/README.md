# Actividad 7: Integración Python + UX (del modelo al producto digital)

## Introducción

Esta actividad tiene como propósito que el estudiante comprenda y ejecute la integración entre el backend computacional desarrollado en Python (modelos de generación y consumo) y el frontend de experiencia de usuario (prototipo UX), construyendo así un producto digital funcional que articule datos, lógica y diseño.

## Objetivo

Integrar los modelos Python desarrollados en actividades anteriores con el prototipo UX diseñado, construyendo un producto digital funcional que conecte el análisis de datos con una interfaz de usuario clara y orientada a la gerencia hospitalaria.

## Desarrollo teórico

La integración backend-frontend es un proceso clave en el ciclo de vida del desarrollo de software. En el contexto del ecosistema hospitalario solar, el backend en Python provee los cálculos de generación, consumo y alertas, mientras que el frontend presenta esta información de manera comprensible para los usuarios. Desde la gerencia de proyectos, esta integración representa la transición del prototipo conceptual al producto digital mínimo viable (MVP).

## Entregables esperados

- Aplicación funcional que integre modelos Python con interfaz de usuario.
- Conexión entre los cálculos de generación/consumo y las visualizaciones del dashboard.
- Pruebas de usabilidad básicas con el producto integrado.
- Documentación de la arquitectura de integración (backend + frontend).

## Conclusión

Esta actividad permite comprender la lógica de integración entre backend y frontend en proyectos digitales sostenibles orientados al sector hospitalario, desarrollando la capacidad de construir productos digitales completos y funcionales.
